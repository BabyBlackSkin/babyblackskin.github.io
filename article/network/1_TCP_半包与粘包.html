TCP粘包和半包问题 ​

什么粘包、什么半包 ​

  • 粘包：所谓的粘包问题就是指在数据传输的时候，在一条消息中读取到了另一条消息的部分数据
  • 拆包：半包是指接收端只收到了部分的数据，而非完整的数据的情况，

为什么会产生粘包与半包 ​

TCP是面向连接的传输协议，它是以流的形式传输数据的，而流数据是没有明确开始和结尾的边界的，所以就会出现粘包和半包问题

实例 ​

使用Java写一个SocketServer

Server

java
package com.babyblack.tcp;

import java.io.IOException;
import java.io.InputStream;
import java.net.ServerSocket;
import java.net.Socket;

/**
 * @author qin.yu.bin
 * @version V1.0.0
 * @ClassName StocketServer
 * @Description
 * @date 2024/7/26 11:50
 */
public class Server {
    private static final int BYTE_LENGTH = 10;

    public static void main(String[] args) throws IOException {
        // 创建 Socket 服务器
        ServerSocket serverSocket = new ServerSocket(8000);
        // 获取客户端连接
        Socket clientSocket = serverSocket.accept();
        // 得到客户端发送的流对象
        try (InputStream inputStream = clientSocket.getInputStream()) {
            while (true) {
                // 循环获取客户端发送的信息
                byte[] bytes = new byte[BYTE_LENGTH];
                // 读取客户端发送的信息
                int count = inputStream.read(bytes, 0, BYTE_LENGTH);
                if (count > 0) {
                    System.out.println("接收到客户端的信息是:" + new String(bytes, 0, count));
                }
            }
        }
    }
}

package com.babyblack.tcp;

import java.io.IOException;
import java.io.OutputStream;
import java.io.PrintWriter;
import java.net.Socket;

/**
 * @author qin.yu.bin
 * @version V1.0.0
 * @ClassName Client
 * @Description
 * @date 2024/7/26 11:51
 */
public class Client {
    public static void main(String[] args) throws IOException {
        String serverAddress = "127.0.0.1";
        int port = 8000;

        try (Socket socket = new Socket(serverAddress, port); PrintWriter out = new PrintWriter(socket.getOutputStream(), true)) {
            String message = "Hello TCP";
            OutputStream outputStream = socket.getOutputStream();
            for (int i = 0; i < 10; i++) {
                outputStream.write(message.getBytes());
            }
        }
    }
}

可以看到出现了 粘包和拆包的情况 解决方法

  • 固定数据大小：发送方和接收方固定发送消息的大小，当字符长度不够的时候用空字符弥补，有了固定大小就知道消息的边界
  • 自定义数据协议（定义数据边界）：在TCP协议的基础上封装上一层自定义数据协议，在自定义的数据协议中，包含数据头（存储数据的大小）和数据的具体内容
  • 以特殊字符结尾：比如以“/n”字符结尾，这样就可以直到数据的具体边界，可以避免粘包问题
