Java的值传递 ​

基本概念

  • 形参&实参
  • 值传递&引用传递

形参&实参 ​

方法的定义可能会用到参数（有参方法），参数在程序语言中分为

  • 实参（实际参数、Arguments）：用于传递给函数/方法的参数，必须有确定的值
  • 形参（形式参数、Parameters）：用于定义函数/方法。接受实参，不需要有确定的值
java
String hello = "Hello!;
// hello 为实参
sayHello(hello);

// str为形参
void sayHello(String str){
    System.out.println(str);   
}

值传递&引用传递 ​

程序设计语言将实参传递给方法（或函数）的方式分为两种：

  • 值传递：方法接收的是实参值的拷贝，会创建副本。
  • 引用传递：方法接收的直接是实参所引用的对象在堆中的地址，不会创建副本，对形参的修改将影响到实参。

案例1：基本类型参数 ​

代码

java
public class Day01 {
    public static void main(String[] args) {
        int num1 = 10;
        int num2 = 20;
        System.out.println("原始值");
        System.out.println("num1 = " + num1);
        System.out.println("num2 = " + num2);

        System.out.println("交换");
        swap(num1, num2);

        System.out.println("Add");
        add(num1, num2);

        System.out.println("操作后");
        System.out.println("num1 = " + num1);
        System.out.println("num2 = " + num2);
    }

    public static void swap(int a, int b) {
        int temp = a;
        a = b;
        b = temp;
        System.out.println("a = " + a);
        System.out.println("b = " + b);
    }


    public static void add(int a, int b) {
        a = a + 10;
        b = b + 10;
        System.out.println("a = " + a);
        System.out.println("b = " + b);
    }
}

输出

java
原始值
num1 = 10
num2 = 20
交换
a = 20
b = 10
Add
a = 20
b = 30
操作后
num1 = 10
num2 = 20

解析 在swap和add方法中，分别对a,b的值进行交换和增加，并不会影响到num1,num2。因为a,b的值，只是从 num1,num2中复制过来的、属于num1,num2的副本，无论副本的内容怎么修改，都不会影响到原件本身

案例2：引用类型参数 ​

代码

java
public class Day02 {
    public static void main(String[] args) {
        int[] arr = {1, 2, 3};

        System.out.println(arr[0]);
        change(arr);
        System.out.println(arr[0]);
    }
    public static void change(int[] a){
        a[0] = 0;
    }
}

输出

java
1
0

这个案例从表面上看觉得Java对引用类型的参数传递是引用传递。 实际并不是，这里传递的是引用类型的地址。 change方法中的形参a是对实参arr地址的拷贝。所以a和arr指向同一个地址值， 也就是同一个数组对象。这也就是为什么change内部对形参的修改实际修改了实参。

案例3 引用类型参数 ​

这个例子更加能说明Java是值传递 代码

java
public class Day03 {
    public static void main(String[] args) {
        int[] arr1 = {1, 2, 3};
        int[] arr2 = {4, 5, 6};

        swap(arr1, arr2);

        System.out.println("arr1:" + Arrays.toString(arr1));
        System.out.println("arr1:" + Arrays.toString(arr2));
    }

    public static void swap(int[] a, int[] b){
        int[] temp = a;
        a = b;
        b = temp;
        System.out.println("a:" + Arrays.toString(a));
        System.out.println("b:" + Arrays.toString(b));
    }
}

输出

java
a:[4, 5, 6]
b:[1, 2, 3]
arr1:[1, 2, 3]
arr1:[4, 5, 6]

此时swap交换后，并没有影响arr1和arr2， 因为在swap中，a和b只是arr1和arr2地址的副本
